Leetcode and Notes

Prefix sum: Technique that can be used on arrays of numbers.
Create an array: prefix prefix[i]: sum of all elements up to the index i (inclusive)

Example:
nums = [5, 2, 1, 6, 3, 8]
prefix = [5, 7, 8, 14, 17, 25]

When a subarray starts at index 0, it is the “prefix” of the array. A prefix sum represents the sum of all prefixes

Prefix sum allows us to find the sum of any subarray in O(1). To find sum of subarray from i to j (inclusive):
prefix[j] - prefix[i - j] or
prefix[j] - prefix[i] + nums[i] (to avoid out of bounds case when i = 0)

Example 1: Given an integer array nums, an array queries where queries[i] = [x, y] and an integer limit, return a boolean array that represents the answer to each query. A query is true if the sum of the subarray from x to y is less than limit, or false otherwise. For example, given nums = [1, 6, 3, 2, 7, 2], queries = [[0, 3], [2, 5], [2, 4]], and limit = 13, the answer is [true, false, true]. For each query, the subarray sums are [12, 14, 12].

Answer: 

def answer_queries(nums, queries, limit):
    prefix = [nums[0]] # initialize prefix
    for i in range(1, len[nums]): # starting at 2nd element, add the value before it and append
        prefix.append(nums[i] + prefix[-1])
    
    ans = []
    for x, y in queries:
        curr = prefix[y] - prefix[x] + nums[x] # find sum from x to y inclusive
        ans.append(curr < limit) # create boolean list

    return ans

Time Complexity: Without prefix sum, O(n) is the worst case, with n being length of nums. If m is the length of queries, that would give O(n+m). With the prefix sum, it costs O(n) to build, but then answering each query is O(1). This gives O(n+m).

Example 2: 2270. Number of Ways to Split Array Given an integer array nums, find the number of ways to split the array into two parts so that the first section has a sum greater than or equal to the sum of the second section. The second section should have at least one number. Brute force: Iterate over each index i from 0 until len(nums)-1. For each index, iterate from 0 to i to find the sum of the left section, and then iterate from i+1 to end to find the sum of the right section. This would give a time complexity of O(n^2).
